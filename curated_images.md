## Part 2: What I Rejected and Why (Discernment)

### Rejected Image 1: AI-Generated "Machine Learning Dashboard"
*   **Description:** A glowing 3D tech dashboard showing complex nodes, flying server icons, and abstract line graphs.
*   **Why it was rejected:** It looked flashy but completely hollow. It was immediately obvious that it was an AI-generated mock rather than actual work. A Lead ML Engineer reviewing my portfolio would instantly recognize it as fluff. Showing the real `pr_curves.png` matplotlib chart may look simple, but it is real, honest data that proves I actually wrote and executed the code.

### Rejected Image 2: AI-Generated "Robot Chatbot Mascot"
*   **Description:** A friendly-looking 3D cartoon robot wearing headphones to represent the AI Agent.
*   **Why it was rejected:** The cartoon aesthetic clashed severely with the minimalist, technical design system. It made the website look like a generic customer-support software page rather than a custom portfolio. I replaced it with a simple robotic terminal emoji (`🤖`) and code block styling to maintain the clean engineering mood.

## Part 3: Real Capture vs. AI Decisions

| **Capstone Project Details** | **Real matplotlib chart (`pr_curves.png`)** | Shows actual model threshold boundaries. A generated line chart would lack genuine axis markings (e.g. baseline comparisons, client-holdout groupings) and lose credibility. |
| **Header Monogram** | **AI-Generated Icon (`favicon_monogram.png`)** | The logo is abstract branding. Using a simple generator ensured exact hex-code color matching and geometric layout without needing complex graphic tools. |
| **AI Agent Avatar** | **Simple emoji (`🤖`)** | Avoids the generic "AI stock image" trap, ensuring the interface remains uncluttered and functional. |
